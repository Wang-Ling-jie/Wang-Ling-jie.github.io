# 计算机组成原理

# 总线的基本概念

# 为什么要使用总线

各部件两两连接（不可行）

# 什么是总线

​ 总线是连接各个部件的信息传输线

​ 是各个部件共享的传输介质（繁忙，瓶颈，结构简单）

# 总线上信息的传送

串行（一位一位）

并行（多位）：多条总线（线之间有干扰，传输距离较短）

# 总线结构的计算机举例

  1. 单总线结构（总线易被占用）

  1. 面向 CPU 的双总线结构（CPU 和主存连专用总线）（主存和外部设备难以通信，需要以 CPU 为媒介）

  2. 以存储器为中心的双总线结构（目前系统总线和存储总线仍然不能同时工作）

# 总线的分类

  1. 片内总线 芯片内部总线
  2. 系统总线 各部件之间的信息传输线
    • 数据总线 双向（数据到 CPU 或 CPU 到地址） 宽度与机器字长、存储字长相关（通常小于机器字长）
    • 地址总线 单向 （CPU 到外部设备） 宽度与存储地址和 I/O 地址有关
    • 控制总线 有出（CPU 向外）（存储器读、存储器写、总线允许、中断确认） 有入（外部设备向 CPU）（中断请求、总线请求）
  3. 通信总线

​ 计算机系统之间或计算机系统和其他系统之间

​ 传输方式：串行、并行

# 总线特性与性能指标

# 总线的物理实现

印刷于主板上，留出接口（插板）

# 总线特性

  1. 机械特性：尺寸、形状、管脚数和排列顺序
  2. 电气特性：传输方向和有效电平范围
  3. 功能特性：每根传输线的功能（地址、数据、控制）
  4. 时间特性：信号的时序关系

# 总线的性能指标

  1. 总线的宽度： 数据线根数
  2. 标准传输率：每秒传输的最大字节数 (MBps)
  3. 时钟同步 / 异步：传输同步、不同步
  4. 总线复用：8086 地址线和数据线共用（20 根地址线，其中 16 条也用于数据线）减小管脚数和芯片体积
  5. 信号线数：地址线、数据线和控制线的总和
  6. 总线控制方式：突发、自动、仲裁、逻辑、计数
  7. 其他指标：负载能力

# 总线标准

如 ISA：每秒传输 8M 次，每次传输 16 位（两字节），故每秒传输 16MB（带宽为 16MBps）

# 总线判优控制

  1. 主设备（模块） 对总线有控制权，可以提出总线占用申请
  2. 从设备（模块） 响应从主设备发来的总线命令

方法：

  1. 集中式（判优逻辑）
    • 链式查询
    • 计数器定时查询
    • 独立请求
  2. 分布式

链式查询方式：

缺点：

  • 优先级低的 I/O 设备提出占用总线的请求可能一直得不到回复
  • 对电路设备故障敏感，一个坏了，后面都不能往下查找
  • 速度比较慢

优点：结构简单，增删设备容易，优先级算法简单，进行可靠性设计容易实现

一般用于微型计算机或简单嵌入式系统

计数器定时查询方式：

总线控制部件中有一个计数器，仍然有 BS，BR，但没有 BG，多了一个设备地址线用于编码地址（设备地址线宽度至少为log2Nlog_2Nlog2​N 向上取整（N 为 I/O 编码设备数））

实现循环查找，且可通过设置计数器自定义优先级，灵活性较强

独立请求方式

总线控制电路内部有一个排队器，确定设备优先级

优先级更灵活，但线更多，N 个设备要 2N 条线

# 总线通信控制

  1. 目的：解决主设备和从设备协调通信问题
  2. 总线传输周期：主设备和从设备之间完成一次完整可靠通信需要时间
  • 申请分配阶段 主模块申请，总线仲裁决定
  • 寻址阶段 主模块向从模块给出地址和命令
  • 传数阶段 主模块和从模块交换数据
  • 结束阶段 主模块撤销有关信息
  1. 总线通信的四种方式
  • 同步通信 有统一时标控制数据传送
  • 异步通信 采用应答方式，没有公共时钟标准
  • 半同步通信 同步异步结合
  • 分离式通信 充分挖掘系统总线每个瞬间的潜力，让总线发挥最大的效能

同步式数据输入

特征：定宽、定距的时标控制传输过程

固定的时间点上给出固定操作

4 个时钟周期

T1T_1T1​ 上升沿：主设备（CPU）给出地址信号

T2T_2T2​ 上升沿：主设备给出读命令信号，告诉从设备要读入数据

T3T_3T3​ 上升沿之前：从设备给出数据信号

T4T_4T4​ 上升沿：撤销数据信号和控制信号

T4T_4T4​ 结束：撤销地址信号

同步式数据输出

T1T_1T1​ 上升沿：主设备（CPU）给出地址信号

T1T_1T1​ 下降沿之前：主设备给出数据

T2T_2T2​ 上升沿：主设备给出写命令信号，向从设备写入数据

T3T_3T3​ 上升沿：正在做写数据的操作

T4T_4T4​ 上升沿：撤销数据信号和控制信号

T4T_4T4​ 结束：撤销地址信号

同步方式：所有从模块都用统一时标，对多个不同模块必须使用最慢的时标。

应用于总线长度比较短，各模块比较一致的通信。

异步通信（请求和回答）

不互锁：主设备不管是否接受到应答信号，经过一定延时后都会撤销请求信号；从设备不管主设备是否接收到应答信号，过一段时间后都会撤销应答信号。（通信可靠性有问题）

半互锁：从设备接受请求并发出应答信号后，当主设备接收到应答信号后才撤销请求，否则主设备保持信号（可能使请求信号一直保持高电平）

全互锁：从设备接受请求并发出应答信号后，发出应答信号。主设备接受到应答信号后才撤销请求；同样，只有主设备信号撤销以后，从设备才会撤销应答信号。（比较可靠）

半同步通信（同步、异步结合）: 允许不同速度的模块数据交换

  • 同步

​ 发送方用系统 时钟前沿 发信号

​ 接收方用系统 时钟后沿 判断、识别

  • 异步

​ 允许不同速度模块和谐工作

​ 增加了一条等待响应信号

以输入数据为例的半同步通信时序

  • T1T_1T1​ 主模块发地址
  • T2T_2T2​ 主模块发命令
  • TwT_wTw​ 当 WAIT 信号为低电平时，等待一个 T
  • TwT_wTw​ 当 WAIT 信号为低电平时，等待一个 T
  • T3T_3T3​ 从模块提供数据
  • T4T_4T4​ 从模块撤销数据，主模块撤销命令

上述三种通信的共同点（以输入数据为例）

从模块准备数据时总线空闲，浪费总线资源

主模块发地址、命令、从模块发送数据占用总线

分离式通信

充分挖掘系统总线每个瞬间的潜力

一个总线传输周期

  • 子周期 1 主模块申请占用总线，使用完后即放弃总线的使用权
  • 子周期 2 从模块申请占用总线，将各种信息传送至总线上

例：读硬盘等待时间长，故通道会断开，不再占用总线，硬盘控制器控制磁盘完成定位操作后向通道发出请求，然后通道发出找扇区的操作，找扇区后硬盘再向通道提出请求，通道再让硬盘传输数据。

特点：

  1. 各模块都有权申请占用总线，从模块主模块可以互相转化
  2. 采用同步方式通信，不等对方回答
  3. 各个模块准备数据时不占用总线
  4. 总线被占用时无空闲，充分利用总线的带宽
